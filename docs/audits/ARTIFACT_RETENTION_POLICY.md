# Phase 23 — Artifact Retention Policy

> Generated: 2026-06-18
> Defines what artifacts should never be committed to git, and how long they may be retained locally.

---

## Policy Statement

Artifacts that are generated, transient, or reproducible from source code SHALL NOT be committed to the repository. Only source artifacts, configuration, documentation, and certified baselines SHALL be version-controlled.

---

## Artifact Classification

### Category A: NEVER COMMIT (Binary Garbage)

These artifacts are generated by runtime processes and provide no value in version control. They must always be gitignored.

| Artifact | Location | Gitignore Pattern | Source |
|----------|----------|-------------------|--------|
| Python bytecode | `__pycache__/` | `__pycache__/` | Runtime |
| Compiled extensions | `*.so`, `*.pyd` | `*.so` | Build |
| Build artifacts | `build/`, `dist/`, `*.egg-info/` | `build/`, `dist/`, `*.egg-info/` | Build |
| Coverage reports | `.coverage`, `coverage.xml`, `htmlcov/` | Various | Test |
| Benchmark runtime output | `benchmarks/baseline.json` | `/benchmarks/baseline.json` | Benchmark |
| Load test output | `benchmarks/load_test_results.json` | `/benchmarks/load_test_results.json` | Load testing |
| Benchmark temp files | `benchmarks/_tmp/` | `/benchmarks/_tmp/` | Benchmark |
| Runtime event logs | `artifacts/*` | `artifacts/` | Operations |
| Soak test snapshots | `artifacts/soak/*` | `artifacts/` | Soak testing |
| Model weight files | `*.pt`, `*.pth` | `*.pt`, `*.pth` | Training |
| Session logs | `session_logs/*` | `session_logs/` | Agent sessions |
| Editor state | `.vscode/`, `.idea/` | Patterns present | Editor |

### Category B: COMMIT (Blessed Baselines Only)

These are reference artifacts that SHOULD be committed after certification.

| Artifact | Location | Policy |
|----------|----------|--------|
| Benchmark reference baseline | `benchmarks/baseline.reference.json` | Commit after blessing with `--bless` |
| Provenance attestations | `results/provenance/` | Commit after release |
| SBOM | `results/sbom/` | Commit after release |
| Trust reports | `results/trust-report/` | Commit after release |
| License compliance | `results/licenses/` | Commit after audit |
| Checksums | `results/checksums.sha256` | Commit after release |

### Category C: REVIEW PERIODICALLY

These are committed but should be reviewed for staleness.

| Artifact | Location | Review Cadence | Archival Policy |
|----------|----------|----------------|-----------------|
| Docs figures | `docs/figures/fig*.png` | Per release | Archive to `docs/archive/manuscript/` after paper acceptance |
| Old RC docs | `docs/releases/RC1_READINESS.md` | Per RC cycle | Archive after next RC certified |
| Historical SBOMs | `results/sbom/sbom.json` | Per release | Keep only latest 3 |

### Category D: ONE-TIME AUDIT OUTPUTS

These are produced by audits and are useful for traceability but generate no ongoing value. The `.gitignore` already covers these with explicit patterns:

- `docs/DUPLICATE_AND_SUPERSEDED_ANALYSIS.md`
- `naming_consistency_assessment.md`
- `CLEANUP_REPORT.md`
- `finding-structured.json`
- `baseline-freeze-session`

---

## Retention Timelines

| Artifact Type | Local Retention | Remote (Git) | Disposal |
|---------------|-----------------|--------------|----------|
| Runtime logs | 30 days | Never | `rm` or log rotation |
| Soak test output | Until next soak | Never | `rm` after review |
| Benchmark output | Until next bless | Never | `rm` after reference update |
| Coverage HTML | Until next test run | Never | `rm` before commit |
| Model checkpoints | Per training run | Never | Manual cleanup |
| Provenance/SBOM | Until next release | Latest only | Archive superseded |
| Audit deliverables | Permanent | Permanent | Move to `docs/audits/` |

---

## Exceptions

Exceptions to this policy require justification via a comment in the `.gitignore` or a pinned issue. Currently no exceptions exist.
